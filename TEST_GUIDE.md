# 🧪 테스트 가이드

## 테스트 전 확인사항

### 1. Firebase 설정 완료 확인
- [ ] Firebase 프로젝트 생성 완료
- [ ] `.env.local` 파일 생성 및 환경 변수 입력 완료
- [ ] Firebase Authentication 활성화 (이메일/비밀번호)
- [ ] Firestore Database 생성 및 보안 규칙 설정 완료

### 2. 개발 서버 실행
```bash
npm run dev
```
브라우저에서 [http://localhost:3000](http://localhost:3000) 접속

## 테스트 시나리오

### 1. 회원가입 및 로그인 테스트

#### 회원가입
1. 홈페이지에서 "회원가입" 클릭
2. 이름, 이메일, 비밀번호 입력
3. "회원가입" 버튼 클릭
4. ✅ 대시보드로 자동 이동 확인

#### 로그인
1. "로그인" 페이지로 이동
2. 등록한 이메일/비밀번호 입력
3. "로그인" 버튼 클릭
4. ✅ 대시보드로 이동 확인

### 2. 책 등록 및 관리 테스트

#### 새 책 추가
1. 대시보드에서 "새 책 추가" 클릭 또는 `/books/new` 접속
2. 책 정보 입력:
   - 제목: "해리포터와 마법사의 돌"
   - 저자: "J.K. 롤링"
   - 총 페이지 수: "320"
   - 현재 읽은 페이지: "0"
3. "책 추가하기" 클릭
4. ✅ 내 서재 페이지로 이동 확인
5. ✅ 추가한 책이 목록에 표시되는지 확인

#### 책 목록 확인
1. `/books` 페이지 접속
2. ✅ 등록한 책이 카드 형태로 표시되는지 확인
3. ✅ 진행률 바가 표시되는지 확인
4. 필터 기능 테스트:
   - "읽는 중" 필터 클릭
   - "완독" 필터 클릭
   - "전체" 필터 클릭

#### 책 상세보기 및 수정
1. 책 카드에서 "상세보기" 클릭
2. ✅ 책 정보가 표시되는지 확인
3. "수정" 버튼 클릭
4. 현재 페이지 수를 변경 (예: 50페이지)
5. "저장" 클릭
6. ✅ 변경사항이 반영되는지 확인

### 3. 독서 기록 테스트

#### 독서 기록 작성
1. 책 상세 페이지에서 "독서 기록하기" 클릭 또는 `/reading-log` 접속
2. 독서 기록 입력:
   - 책 선택: 등록한 책 선택
   - 날짜: 오늘 날짜 (기본값)
   - 읽은 페이지 수: "20"
   - 메모: "오늘 재미있게 읽었다" (선택사항)
3. "기록 저장하기" 클릭
4. ✅ 성공 메시지 확인
5. ✅ 책의 현재 페이지가 업데이트되는지 확인
6. ✅ 대시보드의 통계가 업데이트되는지 확인

#### 연속 독서 일수 확인
1. 대시보드에서 연속 독서 일수 확인
2. 다음 날 다시 독서 기록 작성
3. ✅ 연속 독서 일수가 증가하는지 확인

### 4. 감상문 작성 테스트

#### 감상문 작성
1. 책 상세 페이지에서 "감상문 작성" 클릭 또는 `/reviews/new?bookId=책ID` 접속
2. 감상문 작성:
   - 별점: 5점 클릭
   - 내용: "정말 재미있는 책이었습니다..."
3. "감상문 저장하기" 클릭
4. ✅ 책 상세 페이지로 이동 확인
5. ✅ 경험치가 50 증가했는지 확인 (대시보드에서)

#### 감상문 목록 확인
1. `/reviews` 페이지 접속
2. ✅ 작성한 감상문이 목록에 표시되는지 확인
3. ✅ 별점이 표시되는지 확인

#### 감상문 수정
1. 감상문 목록에서 "수정" 클릭
2. 내용 수정
3. "수정 완료" 클릭
4. ✅ 변경사항이 반영되는지 확인

### 5. 대시보드 통계 테스트

1. `/dashboard` 페이지 접속
2. ✅ 레벨 및 경험치 바 표시 확인
3. ✅ 총 읽은 페이지 수 확인
4. ✅ 완독한 책 수 확인
5. ✅ 연속 독서 일수 확인
6. ✅ 빠른 액션 버튼들이 작동하는지 확인

### 6. 네비게이션 테스트

1. 헤더의 네비게이션 메뉴 클릭:
   - 대시보드
   - 내 서재
   - 통계 (아직 구현 안됨)
   - 업적 (아직 구현 안됨)
   - 랭킹 (아직 구현 안됨)
2. ✅ 각 페이지로 정상 이동하는지 확인
3. 로그아웃 버튼 클릭
4. ✅ 로그인 페이지로 이동하는지 확인

## 예상되는 문제 및 해결방법

### 문제 1: Firebase 연결 오류
**증상**: 콘솔에 Firebase 관련 오류 메시지
**해결**:
- `.env.local` 파일이 프로젝트 루트에 있는지 확인
- 환경 변수 값이 정확한지 확인
- Firebase Console에서 프로젝트가 활성화되어 있는지 확인

### 문제 2: 인증 오류
**증상**: 로그인/회원가입이 작동하지 않음
**해결**:
- Firebase Console > Authentication에서 이메일/비밀번호 인증이 활성화되어 있는지 확인
- Firestore 보안 규칙이 올바르게 설정되어 있는지 확인

### 문제 3: 데이터 저장 안됨
**증상**: 책 추가나 기록 저장이 안됨
**해결**:
- Firestore 보안 규칙 확인
- 브라우저 콘솔에서 오류 메시지 확인
- 네트워크 탭에서 요청이 실패하는지 확인

### 문제 4: 페이지 이동 오류
**증상**: 404 에러 또는 페이지를 찾을 수 없음
**해결**:
- 개발 서버가 실행 중인지 확인 (`npm run dev`)
- URL이 정확한지 확인
- 브라우저 캐시 삭제 후 다시 시도
